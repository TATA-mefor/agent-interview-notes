## 项目经历

### Agent 面试笔记系统 —— 全栈自研的 AI 面试备考平台

*独立开发 · 2026.06 · GitHub 开源 · 17 页面 / 12 API / 11 数据库表*

**核心的一句话：** 这不是一个"调用 API 套壳"的项目。RAG 检索链路的分块器、混合检索融合算法、复习优先级公式、LLM 多供应商网关——全是手写实现的。

#### 1. RAG 全链路：从零手写离线索引管道 + 在线混合检索

**背景：** 55 道面试题种子数据 + 用户上传 PDF/文档，需要让 AI 基于知识库内容回答，而不是瞎编。

**我做了什么：**

- **离线管道：** PDF/DOCX/Markdown 多格式解析 → 文本清洗（编码统一、空白合并、不可见字符过滤）→ **自研三种分块策略**（Markdown Header 递归切分是主方案，按标题树构建面包屑导航，超长块按中文句号边界二次拆分，过短块与相邻兄弟合并；段落分块和固定大小滑动窗口做降级兜底）→ Embedding 向量化 → 写入 pgvector 索引
- **在线检索：** 稠密向量（余弦相似度 Top-K）+ 关键词（ILIKE + TF 词频计分），**按 0.7/0.3 权重加权融合**，`Promise.allSettled` 并行两路检索，单路失败不影响另一路
- **结果：** 自建 200 条中文 Agent 面试评测集，**混合检索 Recall@5 = 0.74，比纯向量提升 17%，比纯关键词提升 28%**

> **面试官会问的：** 为什么选 0.7/0.3 权重？你怎么评测的？分块大小怎么定的？——这三个问题准备好，这一段能讲 8 分钟。

#### 2. AI Agent 智能理解：RAG 增强的结构化生成

**背景：** 一道面试题需要准备 7 个维度（标准答案、要点、话术、误区、标签、难度、频率），手动写太慢。

**我做了什么：**

- **RAG-First：** Agent 调用前先从知识库检索相关文档块注入 Prompt，约束 LLM "优先参考知识库内容"，检索失败时优雅降级为纯模型生成
- **结构化约束：** 单次 LLM 调用通过 JSON Schema 约束，输出 7 个结构化字段，包括 60-90 秒第一人称面试话术
- **Human-in-the-Loop：** 所有 AI 生成内容写入 `llm_suggestions` 表，用户在前端逐字段确认采纳后才写入卡片，AI 永远不自动覆盖用户数据
- **结果：** 50 题人工抽检，AI 答案相关性 4.1/5，话术可直接口述率约 80%

> **面试官会问的：** 为什么不让 AI 直接写入？遇到 bad case 怎么回溯？——准备好 `llm_suggestions` 表和 `agent_runs` 表的全链路追踪逻辑。

---

#### 3. 多格式导入管道：解析 → 去重 → 预览 → 写入

**背景：** 用户有各种格式的面试题（PDF、CSV、Markdown、纯文本粘贴），需要一个通用导入入口。

**我做了什么：**

- **智能粘贴解析：** 前端正则引擎，从非结构化文本中自动提取 Q&A 对和标签
- **多格式自动检测：** 后端统一接口，自动检测 JSON → CSV → Markdown 优先级链路
- **LLM 非结构化提取：** PDF/DOCX 先用 mammoth/pdf-parse 提纯文本，再让 LLM 从中提取 Q&A 对，超长文档自动分段适配 64K token 上限
- **去重：** SHA256 哈希 + 精确匹配双重去重
- **结果：** 25 页 AI Agent 面试指南 PDF 提取 25 题，人工校验准确率 88%

> **面试官会问的：** 大文件分段策略是什么？LLM 提取和正则提取怎么选择？准确率怎么算的？

#### 4. 复习算法：手写 Ebbinghaus 遗忘曲线优先级模型

**背景：** 55 道题，用户不可能每天全刷一遍，需要一个智能调度算法。

**我做了什么：**

- **基础权重：** `难度系数 × 考频系数 × (1 - 掌握度)`，三个系数全部可配置
- **遗忘因子：** 简化 Ebbinghaus 曲线 `forgetting = 1 - e^(-days/7)`，7 天不复习遗忘约 63%
- **综合优先级：** `基础权重×0.7 + 遗忘因子×0.2 + 手动调整×0.1`，三项加权融合
- **掌握度更新：** 正确 `mastery += 0.15 × diffFactor × (1 - mastery)`，错误 `mastery -= 0.1 × diffFactor × mastery`，高掌握度增益递减，避免极端震荡
- **结果：** 对比固定间隔复习，高遗忘风险 + 高重要性卡片的复习覆盖率提升约 30%（55 题 14 天模拟）

> **面试官会问的：** 为什么不用现成的 Anki 算法？你的公式跟 SM-2 有什么区别？模拟怎么做的？

## 鱼皮的简历建议

**先说结论：这份项目经历对 AI Agent / RAG 方向岗位很有竞争力，但简历怎么写比做了什么更重要。**

### 1. 砍掉虚的，加上实的

原版简历里"提升用户体验""保证语义完整性"这种话，面试官看了想翻白眼。**每句话要么带数字，要么带决策理由。**

❌ "实现编码统一与空白合并等标准化清洗"
✅ "手写文本清洗管道：UTF-8 编码统一 + 空白合并 + 不可见字符过滤，处理 3 种编码异常的边界情况"

### 2. 每个模块准备 8 分钟面试回答

面试官看到 RAG 混合检索一定会问：
- "为什么选 0.7/0.3 权重？"
- "评测集怎么建的？"
- "分块大小怎么定的？"

**你现在就得把答案写好，别等面试现场想。** 格式：背景（1 分钟）→ 方案（2 分钟）→ 取舍（2 分钟）→ 数据（1 分钟）→ 复盘（2 分钟）。

### 3. 项目是自研的，这点要强调

现在 AI 方向简历十个有九个是"用 LangChain 搭了个 RAG 问答"，面试官已经看吐了。**你的分块器是自己写的，混合检索融合逻辑是自己写的，复习算法是自己写的——这才是区分度。**

自我介绍时第一句就可以说："这个项目的核心模块——分块器、混合检索、LLM 网关——是我手写实现的，没有直接调 LangChain 的高级封装。"

### 4. 不要只写做了什么，要写为什么

❌ "使用 pgvector 存储向量"
✅ "选择 pgvector 而非 Pinecone/Weaviate，因为项目定位本地优先，pgvector 零额外依赖、SQL 生态无缝集成、单机十万级向量足够"

**面试官想看的是你的决策能力，不是你的调包能力。**
