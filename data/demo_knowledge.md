# Agent 工程面试备考资料

## 一、基础概念

### 什么是 AI Agent？

AI Agent（人工智能代理）是一种能够自主感知环境、做出决策并执行行动以实现特定目标的智能系统。

核心特征：
- **自主性**：无需人类逐步指导，能独立规划和执行任务
- **反应性**：感知环境变化并及时响应
- **主动性**：不仅能被动响应，还能主动采取行动达成目标
- **社会性**：能与其他 Agent 或人类进行交互协作

### Agent 的核心组件

1. **LLM（大语言模型）**：Agent 的"大脑"，负责推理和决策
2. **工具（Tools）**：API 调用、代码执行、数据库查询、网络搜索等
3. **记忆（Memory）**：短期记忆（上下文窗口）、长期记忆（向量数据库）
4. **规划（Planning）**：任务分解、行动计划生成、反思与调整

### Agent 工作模式

**ReAct（Reasoning + Acting）**
- 交替进行推理（Thought）和行动（Action）
- 每次行动后观察结果（Observation）
- 循环直到任务完成
- 优点：推理过程透明可追溯
- 缺点：步骤多导致延迟高

**Plan-and-Execute**
- 先生成完整执行计划，再逐步执行
- 计划与执行可分离到不同模型
- 适合长任务和可并行执行的场景

## 二、核心模块

### Tool Calling（工具调用）

LLM 通过结构化输出调用外部工具的能力。

主流实现：
- OpenAI Function Calling
- Anthropic Tool Use
- 支持并行工具调用

关键设计：
- JSON Schema 定义工具
- 系统提示注入工具描述
- 错误处理与重试机制

### RAG（检索增强生成）

将外部知识库与 LLM 结合，提高回答的准确性和时效性。

Pipeline：
1. 文档解析
2. 文本切分（Chunking）
3. 向量化（Embedding）
4. 存储到向量数据库
5. 查询时检索相关片段
6. 注入 LLM 上下文

关键参数：
- chunk_size：切分大小（通常 500-1500 字符）
- top_k：检索数量（通常 3-10 个）
- similarity_threshold：相似度阈值

### 记忆系统

| 类型 | 存储位置 | 生命周期 | 用途 |
|------|---------|---------|------|
| 感知记忆 | 上下文窗口 | 当前对话 | 最新交互 |
| 短期记忆 | 对话历史 | 当前会话 | 近期上下文 |
| 长期记忆 | 向量数据库 | 跨会话 | 持久化知识 |
| 程序记忆 | 提示词/微调 | 永久 | 技能与行为 |

## 三、架构设计

### 单 Agent vs 多 Agent

单 Agent 适合：
- 任务相对简单
- 工具数量 < 10 个
- 不需要并行处理
- 对延迟要求高

多 Agent 适合：
- 任务复杂需要专业分工
- 需要并行处理提高效率
- 需要相互验证提高可靠性

多 Agent 协调模式：
- **Supervisor**：中央协调者分配任务
- **Peer-to-Peer**：Agent 直接通信
- **Pipeline**：串行流水线处理
- **Blackboard**：共享状态空间

### 错误处理与容错

- 工具调用失败：重试（指数退避）+ 降级
- LLM 输出解析失败：格式修复 + 重试
- 无限循环：最大步骤数限制
- 超时：每个操作设置超时
- Human-in-the-Loop：高风险操作前人工确认

### Prompt Injection 防护

Agent 面临的首要安全威胁。

防护措施：
- 输入验证和清洗
- 系统指令与用户输入严格分离
- 输出验证
- 最小权限原则

## 四、工程实践

### 可观测性

三大支柱：
- **日志（Logs）**：LLM 调用、工具执行、错误
- **指标（Metrics）**：Token 消耗、延迟、成功率
- **追踪（Traces）**：完整执行链路

工具：LangSmith、Langfuse、Helicone

### Token 优化

- 提示词压缩
- 对话历史摘要
- 模型分级（简单任务用小模型）
- Prompt Caching
- 批处理

### 测试策略

- 单元测试：工具函数、提示模板
- 集成测试：LLM + 工具交互
- 端到端测试：完整任务流程
- LLM-as-Judge：自动评估输出质量
- Golden Dataset：标准问答对

## 五、面试常见问题

### Q: Agent 与传统程序的区别？
- 传统程序：预定逻辑，输入输出固定
- Agent：自主决策，动态规划，处理不确定性

### Q: LLM 在 Agent 中的角色？
LLM 是 Agent 的核心推理引擎，负责理解意图、分解任务、选择工具、生成计划。

### Q: 如何评估 Agent 性能？
- 任务完成率
- 步骤效率
- 端到端延迟
- Token 成本
- 输出准确性
